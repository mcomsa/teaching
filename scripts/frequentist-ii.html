1. Fitting frequentist generalized linear multilevel models

Required packages

For generalized linear multilevel models we can use the same packages as for linear multilevel models, lme4:

If you haven’t installed the lme4 package yet, grab the newest version from github. We also require the arm package for predictions:

library("devtools")
install_github("lme4/lme4", dependencies = TRUE) # this can take some time
install.packages("arm")

Then attach the package to make it available in you current session:

library(lme4)
library(arm)
Getting the data

To illustrate generalized linear multilevel modeling with lme4 we investigate turnout in the 2018 US midterm election across demographic, political, and geographic subgroups. For this purpose, we use a small sample from the 2017 voter register of the State of Florida which was complemented with census data. Other than the data we used for linear multilevel modeling, which was strictly nested, these data are cross-classified. While voters are nested in demographic, political, and geographic subgroups, i.e., age, party, district, etc., these subgroups do not nest within each other but are crossed. For example, voters nested in a specific age group are not strictly nested in a specific party affiliation but can share party affiliations with members from other age groups. Mind that this is not a representative sample of the registered voter population as voters with missing information were removed during data preparation and voting-eligibility in terms of active and inactive registrants is not considered. Similarly, the sample is taken from the voter register inbetween the 2016 presidential and the 2018 midterm election, which is another reason why turnout figures diverge from official statistics.

To access the data in R, first specify your working directory via setwd():

setwd("FILE/PATH/TO/YOUR/DIRECTORY")

and then import the data via readRDS (assuming the data is in your working directory, adjust file path if necessary):

voterdat <- readRDS(file = "voterdat.rds")

The data contains the following variables for 2000 observations:

  • vote2018 - turnout in the 2018 general election (outcome)
  • race - respondents’ race/ethnicity (demographic grouping)
  • age - respondents’ age (demographic grouping)
  • party - respondents’ party affiliation (political grouping)
  • income - annual per capita income in respondents’ census block groups rescaled as income/$10,000 (individual-level predictor)
  • congdist - congressional district index (geographic grouping)
  • incumbent - incumbent party before the 2018 US midterm election (geography-level predictor)
Specifying a model

Specifying a generalized linear multilevel model in lme4 works exactly the same way as specifying a linear multilevel model. Have a look at the ‘frequentist-lm.html’ if you need to catch up on this.

Fitting generalized linear multilevel models

Use lme4’s glmer() (generalized linear mixed-effects regression) function to fit linear multilevel models. The glmer() function takes three central arguments:

  • the formula argument is specified as for linear multilevel models
  • the data argument takes the data.frame with the variables
  • the family argument specifies the appropriate error distribution and model link function.

Note that the REML argument does not exist for glmer() as it is not clear how the bias correction of ML estimators achieved by REML generalizes to generalized linear multilevel models (see here). R packages such as glmmTMB and hglm, however, have implemented REML-like estimation. The glmmTMB package should also be considered for more complex scenarios that require zero-inflated or hurdle models, which are not covered by glmer().

As an example of a glmer() workflow we focus on the binomial family and the logit link function, i.e., a multilevel logistic regression. As for notation, given a binary outcome, \(y_i\) now follows a binomial distribution, whereby \(n_i\) is \(1\) for all \(i\) (hence reducing to a bernoulli distribution) and \(\pi_i\) is the deterministic component transformed by the provided link function to yield probabilities between \(0\) and \(1\). Otherwise we proceed as for linear multilevel models and include the global/population/fixed intercept in the regression model for \(y\) and center the distribution for the group-level models at 0. This allows for interpretations of the group-level parameters as deviations from the population intercept. It further aligns with the lme4 output provided via the fixef() and ranef() functions and conceputally simplifies the construction of the linear predictor with multiple variables for grouping.

Let’s begin with a varying-intercept model without predictors:

\(y_i \sim\) Bin\((n_i, \pi_i)\)
\(\pi_i =\) logit\(^{-1}(\mu + \alpha_{j})\), for \(i = 1, ..., n\)
\(\alpha_j \sim\) N\((0, \sigma^2_\alpha)\) for \(j = 1, ..., J\)

where \(\mu\) is the population intercept, \(\alpha_{j}\) is a varying-intercept at race groups \(J\). Note that the distribution for the demographic grouping model is centered at 0 with \(\sigma^2_\alpha\) denoting the group-level variance. In consequence, \(\alpha_{j}\) is the deviation of the race-specific intercepts from the population intercept \(\mu\) so that \(\mu\) + \(\alpha_{j}\) provides the race-specific intercepts. To estimate this model in R, type:

glm_1 <- glmer(formula = vote2018 ~ 1 + (1 | race), 
               data = voterdat, 
               family = binomial(link = "logit"))
#> Warning: 'rBind' is deprecated.
#>  Since R version 3.2.0, base's rbind() should work fine with S4 objects

We will have a look at the fitted models and how to interpret them below, for now we learn how to estimate them. Next up is a varying-intercept model with an individual-level predictor included in the regression model for \(\pi_i\):

\(y_i \sim\) Bin\((n_i, \pi_i)\)
\(\pi_i =\) logit\(^{-1}(\mu + \alpha_{j} + \beta x_i)\), for \(i = 1, ..., n\)
\(\alpha_j \sim\) N\((0, \sigma^2_\alpha)\) for \(j = 1, ..., J\)

where \(\beta\) represents the slope of \(x_i\), a person’s income, which does not vary by race. To estimate this model in R, type:

glm_2 <- glmer(formula = vote2018 ~ income + (1 | race),
               data = voterdat,
               family = binomial(link = "logit"))

In a next step, age, party, and congressional district are introduced as additional demographic, political and geographic grouping levels. As these groups are crossed, their models do no inform each other as was the case in the strictly hierarchical linear multilevel model that we covered before.

\(y_i \sim\) Bin\((n_i, \pi_i)\)
\(\pi_i =\) logit\(^{-1}(\mu + \alpha_{j} + \gamma_{k} + \delta_{l} + \zeta_{m} + \beta x_i)\), for \(i = 1, ..., n\)
\(\alpha_j \sim\) N\((0, \sigma^2_\alpha)\) for \(j = 1, ..., J\)
\(\gamma_k \sim\) N\((0, \sigma^2_\gamma)\) for \(j = 1, ..., K\)
\(\delta_l \sim\) N\((0, \sigma^2_\delta)\) for \(j = 1, ..., L\)
\(\zeta_m \sim\) N\((0, \sigma^2_\zeta)\) for \(j = 1, ..., M\)

where \(\gamma_k, \delta_l, \zeta_m\) are varying-intercepts at age, party, and congressional district groups respectively and \(\sigma^2_\gamma, \sigma^2_\delta, \sigma^2_\zeta\) denote the respective group-level variances. Instead of having many different greek letters, we might distinguish the varying intercepts by writing \(\alpha^{race}_{j}, \alpha^{age}_{k},\) etc. and \(\sigma^{2}_{race}, \sigma^{2}_{age},\) etc. The choice of notation is a matter of taste.

glm_3 <- glmer(formula = vote2018 ~ income + (1 | race) + (1 | age) + (1 | party) + (1 | congdist),
               data = voterdat,
               family = binomial(link = "logit"))

Now we add a group-level predictor to the model, the incumbent party before the 2018 US midterm election at the congressional district level:

\(y_i \sim\) Bin\((n_i, \pi_i)\)
\(\pi_i =\) logit\(^{-1}(\mu + \alpha_{j} + \gamma_{k} + \delta_{l} + \zeta_{m} + \beta x_i)\), for \(i = 1, ..., n\)
\(\alpha_j \sim\) N\((0, \sigma^2_\alpha)\) for \(j = 1, ..., J\)
\(\gamma_k \sim\) N\((0, \sigma^2_\gamma)\) for \(j = 1, ..., K\)
\(\delta_l \sim\) N\((0, \sigma^2_\delta)\) for \(j = 1, ..., L\)
\(\zeta_m \sim\) N\((\beta_1 u_m, \sigma^2_\zeta)\) for \(j = 1, ..., M\)

where \(\beta_1\) is the slope of the cogressional district-level predictor \(u_m\), the incumbent party before the 2018 election.

glm_4 <- glmer(formula = vote2018 ~ income + incumbent + (1 | race) + (1 | age) + (1 | party) + (1 | congdist),
             data = voterdat,
             family = binomial(link = "logit"))

Finally, we add one more complication and let the \(\beta\), the slope of \(x_i\), a person’s income, vary across race groups:

\(y_i \sim\) Bin\((n_i, \pi_i)\)
\(\pi_i =\) logit\(^{-1}(\mu_\alpha + \mu_\beta x_i + \alpha_{j} + \gamma_{k} + \delta_{l} + \zeta_{m} + \beta_{j} x_i)\), for \(i = 1, ..., n\)
\(\alpha_j \sim\) N\((0, \sigma^2_\alpha)\) for \(j = 1, ..., J\)
\(\gamma_k \sim\) N\((0, \sigma^2_\gamma)\) for \(j = 1, ..., K\)
\(\delta_l \sim\) N\((0, \sigma^2_\delta)\) for \(j = 1, ..., L\)
\(\zeta_m \sim\) N\((\beta_1 u_m, \sigma^2_\zeta)\) for \(j = 1, ..., M\)
\(\beta_{j} \sim\) N\((0, \sigma^2_\beta)\) for \(j = 1, ..., J\)

where \(\mu_\beta\) is the population slope/fixed effect of \(x_i\), a person’s income. Before, this was \(\beta\), but this slope varies across race groups now, \(\beta_{j}\) is therefore the deviation of the race-specific slopes from the population slope \(\mu_\beta\). The population intercept \(\mu\) is now \(\mu_\alpha\) to distinguish it from \(\mu_\beta\).

glm_5 <- glmer(formula = vote2018 ~ income + incumbent + (1 + income | race) + (1 | age) + (1 | party) + (1 | congdist),
             data = voterdat,
             family = binomial(link = "logit"))

3. Basic interpretation

While the summary output for glmer() is structured in basically the same way as for lmer() and looks very similar, the interpretation of the coefficients is not as intuitive. This is because the coefficients returned by glmer() are continuous values, in our case on the logistic scale. Furthermore, differences in predicted outcome values depend on the value of the predictor and are not constant across the full range of the predictor variable as is the case for linear models. Hence, for generalized linear multilevel models only the direction/sign of the coeffcients and the magnitude of group-level coefficients relative to each other are intuitively interpretable. For interpretations on probability scale, we have to transform the linear predictor using the inverse of the link function, in our case the inverse logistic function.

We again begin by extracting different parts of the output from the glmer() function:

Using the fixef() function, we can extract the population- and group-level estimates from the model:

#>  -0.1513315

#> 0.0847454

To arrive at predictions of turnout probabilities for individuals in specific demographic, political, and geographic subgroups and with specific values at individual and group-level predictors, we can replicate the linear predictor accordingly and transform it using the inverse logistic function provided by invlogit from the arm package.

## predictions based on a model with the intercept varying by race and a person's income as individual level predictor

# predicted turnout probability for a hispanic voter
invlogit( # inverse logistic tranformation
  fixef(glm_1)["(Intercept)"] + # population intercept
  ranef(glm_1)$race[2,] # varying intercept
)
#> (Intercept) 
#>    0.482941

# for a black voter with the same income
invlogit(
  fixef(glm_1)["(Intercept)"] + 
  ranef(glm_1)$race[1,]
)
#> (Intercept) 
#>   0.5401574

# for a white voter with the same income
invlogit(
  fixef(glm_1)["(Intercept)"] + 
  ranef(glm_1)$race[4,]
)
#> (Intercept) 
#>   0.5460085


# predicted turnout probability for a hispanic voter from a neighborhood with an annual per capita income of 15,000
invlogit(
  fixef(glm_2)["(Intercept)"] + # population intercept
    fixef(glm_2)["income"] * 1.5 + # population slope
    ranef(glm_2)$race[2,] # varying intercept 
)
#> (Intercept) 
#>   0.4643176

# for a black voter with the same income
invlogit(
  fixef(glm_2)["(Intercept)"] + 
    fixef(glm_2)["income"] * 1.5 +
  ranef(glm_2)$race[1,]
)
#> (Intercept) 
#>   0.5145614

# for a white voter with the same income
invlogit(
  fixef(glm_2)["(Intercept)"] + 
  fixef(glm_2)["income"] * 1.5 +
  ranef(glm_2)$race[4,]
)
#> (Intercept) 
#>    0.505794

  ranef(glm_3)$race[2,] + # varying intercepts

Again, these examples are interesting and useful to explore and understand the fitted model. But constructing these linear predictors for different individuals is also quite tedious. Usually we are interested in predictions that summarize the model in a more comprehensive fashion, i.e., a population-averaged interpretation of estimates, that is, estimates of the average outcome in the population given specific levels of input variables as opposed to predictions for very specific observations. We will learn how to obtain and communicate population-averaged predictions and their uncertainty for generalized linear multilevel models in the sessions on ‘presenting results of multilevel models’ using simulation (or informal Bayesian inference).
